1 Start with HTML

1.1 The basics of Web and HTML

Definition

HTML = Hyper Text Markup Language.

Text Content : What we see.
Markup : Formating.
References & links : Images, videos, links to other pages...

It's made of elements, with an oppening Tag, Content, Closing Tag. The tag can embed attributes with values, like for an image : the source (src) and alternative content (alt).

Inline/Bloc

Inline elements are like a line of text :

  • span
  • br
  • em
  • img
  • a
  • i
  • strong
  • em

Blocs have a width and a heigth :

  • div
  • h1
  • h2...
  • p
  • form

An element can be switched to inline/bloc with css property (to confirm ?)

1.2 Structured document

DOM

Document Object Model. Means the page should have a tree-like structure. This structure is made by the html. The css style it.

div and span are the two main container to build the structure, but i also can use footer, header, article, section, nav, aside...

Boxify

Everythink is a BOX ! Boxify a layout by organising the page with main boxes organised themeselfes with other boxes and boxes inside. Boxify with pencil, sicors, photoshop...

Code > Test > Refine

Normal workflow :

  1. Boxify the design
  2. Construct with semantic elements (h1, h2, article...)
  3. Write html
  4. Style it with CSS, big to small parts
  5. Fix ! Go back html to css...

1.3 Adding CSS Style

Cascading Style Sheet

The styles applied to an element is applied to every child element. They got "inherited" style for their parents.
That applied for text, not blocs (inline/bloc ?)

Tags already contains some styles. For exemple, h1 is 2em, bold, margin 0,67em... It can be change, but it's use the best tag at first.

Avoid repetition

It's good to avaid repetition when programming. If I want a style be applied for several elements, I should create a class ! I also can create several classes and mix them with the shared an non-shared properties (cf trafic lights : one commun class for make them round, one for each for the colors).

Selectors

There's many selectors for styling. The more specific will affect instead of the less specific. The list is ordered by specificity :

  • !important place just after the declaration overide anythink. Use in extrame case of need.
  • inline style is also very powerfull and no recommanded because it mix structure and style.
  • ID is specific and can be use only once.
  • Class / Attribute are the most recomanded. Class is well know, but attribute is better when the attribute is enought specifical because you don't have to write a class (avoid repetition).
  • Element tags like h1, div, img, ul... Good to use in addition with classes.

You can combine selectors. The style will apply to elements validating all of them. See this CSS tricks post for selectors generalities.

Boxes strucure

A bloc element (and inline ?) has a size defined by :

  • Content
  • Padding (space around content)
  • Border (border weight)
  • Margin (space around border)

If I have a padding and border, i'ts hard to know and manage the final size of the box. I can use the css propertie {box-sizing : border-box;}. Warning : use webkit, moz, ms... for compatibility. It's also recomanded to use percentage (%) when possible.

Flex-Box

Blocs elements are 100% widht by default. I can allow them to align side by side with the propertie {display : flexbox} applied to the parent element. Don't foget to reduce the size of child smaller than 100% (80%, max-width, ...). A complete guide to floxbox in css-tricks.
Check compatibility !
